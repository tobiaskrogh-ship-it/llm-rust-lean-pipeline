The available `natChoose_le_two_pow` bound is `2^n` which exceeds 2^64 for n ≥ 64, blocking the body step of `binomial_loop_case`. We need a tighter bound `natChoose n k &lt; 2^64` for n ≤ 67 (which holds since natChoose 67 33 &lt; 2^64). Adding this as a new helper via native_decide on the finite table.
