theorem gcd_while_total (a b : u64) :
    ∃ r : u64, gcd_while.gcd_while a b = RustM.ok r := by
  -- Existence of an `RustM.ok` result requires excluding both panic and
  -- divergence. The Hax `while_loop.spec` lemma gives a Hoare triple, which
  -- is *insufficient* on its own to derive `∃ r, p = .ok r` because the
  -- triple's failure post defaults to vacuously True (the WP framework
  -- allows `.fail`/`.div`). To rigorously pin existence to the `.ok` case,
  -- the proof would need to combine the loop spec (no panic from the cond
  -- guard `b !=? 0` discharging `a %? b`) with the termination measure
  -- `b.toNat` (excluding `.div`). The library has no precedent for this
  -- pattern for `rust_primitives.hax.while_loop`; left as `sorry`.
  sorry
