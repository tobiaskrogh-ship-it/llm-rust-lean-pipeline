/-- Stein's binary GCD correctness for `gcd_lcm_u64.gcd`.

    **Progress in this pass**:
    1. Zero-input branches (`x = 0` or `y = 0`): closed inline. The inner
       `if (m == 0 || n == 0)` short-circuits and the function returns
       `m ||| n` without entering any loop.
    2. Both-nonzero branch: opened up. After `unfold gcd_lcm_u64.gcd` and
       `simp only` reducing the outer if to its else branch via `hxbeq`
       and `hybeq` (both proving `(x == 0) = false`, `(y == 0) = false`),
       the residual goal exposes the inner structure: three
       `trailing_zeros_u64` calls (each itself a `rust_primitives.hax.while_loop`
       on a `⟨count, y⟩` state), an outer Stein's subtract-and-shift
       `while_loop`, and a final `m <<<? shift`.

    **Specific stuck sub-goal in this pass (the surviving `sorry`)**:
    After the simp-only collapse above, the goal is

    ```lean
    (do
      let shift ← trailing_zeros_u64 (x ||| y)
      let m ← x >>>? (← trailing_zeros_u64 x)
      let n ← y >>>? (← trailing_zeros_u64 y)
      let ⟨m, n⟩ ← rust_primitives.hax.while_loop … ⟨m,n⟩ … (subtract-and-shift body)
      m <<<? shift)
    = RustM.ok (UInt64.ofNat (Nat.gcd x.toNat y.toNat))
    ```

    This needs five chained Hoare-triple proofs (four `trailing_zeros_u64`
    calls, one outer Stein's loop), each with its own invariant and
    termination measure, then assembled via `Triple.bind` and converted to
    the equation form via `RustM.Triple_iff_BitVec` (the canonical two-stage
    pattern from `while_example/README.md`). The Nat-level reduction
    helpers `nat_gcd_sub_left_eq`, `nat_gcd_sub_right_eq`, `nat_gcd_two_mul`,
    `nat_gcd_two_mul_odd`, and `nat_gcd_pow_two_mul` (defined above) are the
    body-step facts each loop will consume.

    **Structural unblock**: a separately-proved
    `proof_patterns/stein_gcd_u64` pattern (targeting exactly this nested-
    loop shape) would close the remaining sorry. Specifically, it needs to
    add to the proof library:
    - `trailing_zeros_u64_triple : ⦃⌜x ≠ 0⌝⦄ trailing_zeros_u64 x
        ⦃⇓ r => ⌜x.toNat = (x.toNat >>> r.toNat) * 2 ^ r.toNat ∧ …⌝⦄`
      with invariant on `⟨count, y⟩` that `count.toNat ≤ 63` and
      `x.toNat = y.toNat * 2 ^ count.toNat`.
    - `gcd_inner_loop_triple` with invariant
      `Nat.gcd m.toNat n.toNat = Nat.gcd (x.toNat >>> tz x.toNat) (y.toNat >>> tz y.toNat)`
      and termination measure `m.toNat + n.toNat`.
    - The closing assembly via `nat_gcd_pow_two_mul` (above) for the
      `m <<< shift` final step. -/
