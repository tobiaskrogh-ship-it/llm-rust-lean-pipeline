/-- Helper for totality: under the invariant
    `acc.val.size = if i.toNat = 0 then 0 else 2 * i.toNat - 1`,
    `intersperse_at` returns `ok v` for some `v`.

    **Stuck sub-goal.** In the `i > 0` branch of the strong induction we
    need to apply `intersperse_at_step_pos`, whose `h_no` precondition is
    `acc.val.size + 2 < 2 ^ 64`. Substituting the invariant
    `acc.val.size = 2 * i.toNat - 1` reduces this to
    `2 * i.toNat + 1 < 2 ^ 64`. From the available hypotheses
    (`i.toNat < numbers.val.size`, `numbers.val.size < 2 ^ 64`) this is
    *not* derivable in the regime `2 ^ 63 ≤ numbers.val.size < 2 ^ 64` —
    pick `i.toNat = 2 ^ 63 - 1` to falsify `2 * i.toNat + 1 < 2 ^ 64`.

    **Structural unblock.** The cleanest external change would be a Hax
    prelude lemma `RustSlice.double_size_le_usizeSize :
    ∀ (s : RustSlice α), 2 * s.val.size ≤ USize64.size`. Such a lemma is
    obviously false (a `RustSlice` of length `2^63` falsifies it), so it
    can only be obtained by tightening the `RustSlice` invariant from
    `val.size < USize64.size` to `2 * val.size ≤ USize64.size`. A more
    realistic unblock: add the precondition
    `h_size_bound : 2 * numbers.val.size ≤ 2 ^ 64`
    to `intersperse_total` (and propagate it through `intersperse_at_total_aux`).
    Under that precondition, the stuck sub-goal becomes
    `2 * i.toNat + 1 ≤ 2 * (numbers.val.size - 1) + 1 = 2 * numbers.val.size - 1
       < 2 * numbers.val.size ≤ 2 ^ 64`, which discharges by `omega`. The
    Rust proptest in `src/lib.rs` restricts lengths to `0..32`, so the
    precondition is satisfied in every realistic execution. -/
