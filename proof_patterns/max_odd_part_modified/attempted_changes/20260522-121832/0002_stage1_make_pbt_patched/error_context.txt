The existing test suite covers the bound (`<= n`) and oddness postconditions but does not pin down maximality — the "largest" part of "largest odd part among 1..=n". A buggy implementation returning a constant `1` for all `n >= 1` would pass `result_at_most_n` and `result_is_odd_when_nonempty`. Adding two independent property tests (achievability + upper-bound) to capture the maximality clause.
