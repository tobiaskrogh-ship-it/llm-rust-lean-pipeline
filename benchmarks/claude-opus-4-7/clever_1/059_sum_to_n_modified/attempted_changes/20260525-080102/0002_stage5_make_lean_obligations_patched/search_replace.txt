namespace Clever_047_sum_to_nObligations

/-! ## Boundary clause: non-positive inputs collapse to zero.

Captures the property test `non_positive_inputs_yield_zero`
(`for n in -5..=0: sum_to_n(n) == 0`). The natural universal Lean
generalisation — every `n : i64` with `n.toInt < 1` returns `ok 0` —
holds without further precondition: the function takes the `n <? 1`
branch and returns `pure 0` directly, no recursion, no arithmetic. -/
theorem sum_to_n_non_positive (n : i64) (h : n.toInt < 1) :
    clever_047_sum_to_n.sum_to_n n = RustM.ok (0 : i64) := by
  sorry

/-! ## Closed-form postcondition

Captures the property test `matches_closed_form`
(`for 1 ≤ n ≤ 5_000: sum_to_n(n) == n*(n+1)/2`). The proptest's `n ≤ 5_000`
bound is a *test* bound that keeps the Rust recursion depth small; the
true model-level constraint is that the running accumulator `acc + k`
must not overflow `i64` at any point.

The maximum accumulator value reached by `sum_to_n_at n 1 0` is
`n*(n+1)/2` (the final returned value), and the sequence is monotonically
increasing through naturals while `1 ≤ k ≤ n`. So the no-overflow
precondition is `n*(n+1)/2 < 2^63`, which subsumes overflow of every
intermediate `acc +? k` step and of the counter `k +? 1`
(`k ≤ n + 1 ≤ n*(n+1)/2 + 1 < 2^63` for `n ≥ 1`).

We use an existential return form (rather than committing to a specific
`Int64.ofInt` encoding) so the obligation reads naturally as "the
function succeeds and the result's `toInt` equals the Gauss closed form".
-/
theorem sum_to_n_closed_form (n : i64)
    (hpos : 1 ≤ n.toInt)
    (hfit : n.toInt * (n.toInt + 1) / 2 < 2 ^ 63) :
    ∃ r : i64,
      clever_047_sum_to_n.sum_to_n n = RustM.ok r ∧
      r.toInt = n.toInt * (n.toInt + 1) / 2 := by
  sorry

/-! ## Failure condition: signed-overflow on the closed-form value.

When the Gauss closed-form sum `n*(n+1)/2` exceeds `i64`'s positive
range, the running accumulator `acc + k` overflows at some step (the
first `k` whose `k*(k+1)/2 ≥ 2^63`), and the function panics with
`Error.integerOverflow`. This pins down the boundary of the
no-overflow precondition above: for `n ≥ 1`, the function is total
*iff* `n*(n+1)/2 < 2^63`.

Not directly tested by the Rust property tests (the proptest is bounded
to `n ≤ 5_000`, well inside the safe range), but mirrored from the
`sum_to_n_modified` reference where the analogous u64 overflow theorem
is one of the two top-level obligations. -/
theorem sum_to_n_overflow (n : i64)
    (hpos : 1 ≤ n.toInt)
    (hov  : 2 ^ 63 ≤ n.toInt * (n.toInt + 1) / 2) :
    clever_047_sum_to_n.sum_to_n n = RustM.fail .integerOverflow := by
  sorry

end Clever_047_sum_to_nObligations
