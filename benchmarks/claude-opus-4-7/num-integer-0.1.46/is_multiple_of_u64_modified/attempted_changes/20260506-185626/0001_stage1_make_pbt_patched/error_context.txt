Existing test suite captures C1/C2 (b==0 special case) but C3/C4 (b>0 divisibility) are only covered tautologically (`a % b == 0` mirrors the implementation) and only on a small range (0..=50, 1..=20). Adding witness-based tests for C3 (constructive multiples a = k*b) and C4 (constructive non-multiples a = q*b + r) so the suite directly checks the existential contract rather than the implementation against itself, plus a large-values test to exercise the upper end of u64.
