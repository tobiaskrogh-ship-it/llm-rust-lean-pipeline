    **Extraction infidelity at `align = 0` (one `sorry`, isolated).** In real
    Rust, `is_power_of_two_usize(0)` short-circuits on `0 != 0` and returns
    `false`, so the function returns `usize::MAX`. The Hax Lean model of
    Rust's short-circuit `&&` is *strict*: `&&?` is
    `rust_primitives.hax.logical_op.and = pure (a && b)`, an ordinary strict
    `Bool` function (verified in `Hax/rust_primitives/hax/logical_op.lean`,
    lines 11 and 25), and the surrounding `do`-block hoists `(← (n -? 1))`
    unconditionally — so `is_power_of_two_usize (0:usize)` **panics** at the
    `0 -? 1` underflow (machine-checked: `ipo2_zero`). The panic propagates
    through the monadic bind, giving
    `padding_needed_for size 0 = RustM.fail Error.integerOverflow`
    (machine-checked: `padding_needed_for_zero_panics`).

    **Stuck sub-goal (this attempt, not inherited).** A substantive attempt
    was made — `rw [padding_needed_for_zero_panics]` then
    `simp only [RustM.fail, RustM.ok]; decide` — and `lake build` rejected it
    with: *“Tactic `decide` proved that the proposition
    `some (Except.error Error.integerOverflow) = some (Except.ok 18446744073709551615)`
    is false.”* (`RustM = ExceptT Error Option`, so `RustM.fail e =
    some (.error e)` and `RustM.ok v = some (.ok v)` are distinct `Except`
    constructors under `some`.) The goal is therefore not merely hard — it is
    *decidably false* — and the hypothesis `h : ¬ IsPow2 align` specialises at
    `align = 0` to the true, leverage-free `¬ IsPow2 0` (`IsPow2 0` is itself
    false), so no contradiction is available to discharge the branch. This is
    confirmed independently by the sorry-free `padding_needed_for_zero_ne_max`.

    **Structural unblock (not a tactic).** This is an upstream extraction
    issue: the Hax backend must emit a short-circuiting / lazy encoding for
    Rust's `&&` (e.g. a `RustM`-level `if` or a lazy boolean-bind whose right
    operand's effects are sequenced only when the left operand is `true`) so
    that `is_power_of_two_usize 0` evaluates to `RustM.ok false` instead of
    panicking. Absent that, the obligation must exclude `align = 0`, since the
    extracted function genuinely diverges from real Rust there. The
    `align ≠ 0` case (the entire rest of this contract clause) is proved in
    full below. -/
